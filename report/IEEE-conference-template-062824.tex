\section{Introduction}
Voxelization is the process of ``discretizing'' a 3D object into an
axis-aligned grid. Besides being used stylistically in video games, this kind
of representation is also used in many fields, such as medical imaging, urban
analysis, computer graphics, and simulations. This report explores the design
process of a parallel voxelization algorithm, which is implemented in C++ and
utilizes the OpenMP library, taking advantage of multi-core processing to
enhance performance.

\section{Development}
\subsection{Algorithm Description}
The goal of the algorithm is to generate a representation of the 3D object as a
set of voxels (volumetric pixels). A volumetric resolution is defined, which
determines the number of voxels used along each axis.

The main idea is to iterate through the model's triangles and paint the voxels
they intersect with. There are several ways to determine this voxel set, and a
quick-to-implement approach is to check all voxels inside a triangle's bounding
box.

\subsection{Paradigm Choice}
This voxelization algorithm executes the same ``check-and-paint'' procedure for
each triangle, which makes it a good candidate for the task parallelization
paradigm. The model's triangles are distributed among the available processes,
checked, and then used to paint the corresponding voxels. Each local result can
then be merged to produde the voxelized output.

\subsection{PRAM}
The theoretical PRAM model is used to build a high-level version of the
algorithm, which considers infinite processes. The algorithm is as follows:

\begin{algorithm}
    \caption{Voxelization (PRAM)}
    \begin{algorithmic}[1]
        \State{} $triangles \gets \text{triangulate}(\text{model.load}(path)).faces$
        \ForAll{triangle \textbf{in} triangles \textbf{pardo}}
        \ForAll{voxel \textbf{inside} bounding box of triangle}
        \State{} paint voxel
        \EndFor{}
        \EndFor{}
        \State{} rebuild 3D model using voxels
    \end{algorithmic}
\end{algorithm}
